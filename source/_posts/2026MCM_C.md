## 摘要

本研究基于 *Dancing with the Stars*（与星共舞）34 个赛季的数据，构建了一个贝叶斯 Markov Chain Monte Carlo (MCMC) 逆优化模型以重构未公开的粉丝投票隐分布，每个淘汰事件执行 $N = 50{,}000$ 次 Monte Carlo 采样以确保统计稳健性。模型将 Rank-based 与 Percentage-based 两种投票聚合机制映射到统一的连续隐人气尺度，实现了跨时代的标准化比较。

研究表明，当选手的相对人气超过周均值的 $+1.0\sigma$ 时将触发"粉丝救援"效应——粉丝支持在决定生存概率时显著超越技术评分。反事实政策模拟显示，Percentage 系统在结构上放大了粉丝团体的极化效应（如 Bobby Bones 的 $\approx 26.7\%$ 投票份额可完全抵消低评委分），而 Rank 系统通过"排名压缩"对极端选手的优势形成了有界的约束。Judge Save 机制仅在"边际争议"案例中有效（如 Billy Ray Cyrus），无法改变超级人气选手的晋级结果。通过多元 Logistic 回归与 OLS 回归，本研究量化了年龄（评委线性递减 $\beta = -0.40$ vs. 粉丝 U 形峰值 49.7 岁）、行业背景（社交媒体红人粉丝系数 $+2.53$，为其技术系数的 $6\times$）和职业舞伴等因素的差异化影响。基于多目标网格搜索优化，提出**动态分阶段投票策略**：Phase 1 (Week 1–8) 以 $w_{\text{judge}} = 0.10$ 保障观众留存，Phase 2 (Week 9–决赛) 调整为 $w_{\text{judge}} = 0.45$ 以兼顾技术公平，辅以 Technical Safety Net，将技术选手误淘汰率 (FER) 从 $5.2\%$ 降至 $0.0\%$，人气选手留存率 (RR) 提升至 $91.8\%$。

**关键词：** 贝叶斯蒙特卡洛模拟 · 逻辑回归 · 竞赛公平性

## 1. 问题背景

*Dancing with the Stars* 是一档源自英国的国际舞蹈竞技真人秀节目，美国版已完成 34 季。其核心淘汰机制定义如下：

> **定义 1 (淘汰规则).** 设第 $t$ 周剩余 $K$ 名选手。对每位选手 $i \in \{1,\ldots,K\}$，已知评委分向量 $\mathbf{S}^{(t)} = (S_1^{(t)},\ldots,S_K^{(t)})$ 与未知粉丝票向量 $\mathbf{V}^{(t)} = (V_1^{(t)},\ldots,V_K^{(t)})$。给定聚合函数 $f: \mathbb{R}^K \times \mathbb{R}^K \to \mathbb{R}^K$，组合得分向量为 $\mathbf{C}^{(t)} = f(\mathbf{S}^{(t)}, \mathbf{V}^{(t)})$，淘汰选手 $k$ 满足 $k = \arg\max_i C_i^{(t)}$ 或 $k = \arg\min_i C_i^{(t)}$，取决于聚合函数的定义。

节目历史上采用了两类聚合函数：

**排名求和聚合 (Rank Aggregation)** — 适用于 Season 1–2, 28–34：

$$
\boxed{C_i^{(t)} = R_J(S_i^{(t)}) + R_F(V_i^{(t)})} \tag1
$$

其中 $R_J(\cdot)$ 为评委分降序排名（最高分 $\mapsto$ Rank 1），$R_F(\cdot)$ 为粉丝票降序排名。淘汰条件为 $k = \arg\max_i C_i^{(t)}$；若存在平局，粉丝排名更差（数值更大）者出局。

**百分比求和聚合 (Percentage Aggregation)** — 适用于 Season 3–27：

$$
\boxed{C_i^{(t)} = P_{J,i}^{(t)} + P_{F,i}^{(t)} = \frac{S_i^{(t)}}{\sum_{j=1}^{K} S_j^{(t)}} + \frac{V_i^{(t)}}{\sum_{j=1}^{K} V_j^{(t)}}} \tag2
$$

淘汰条件为 $k = \arg\min_i C_i^{(t)}$；若存在平局，粉丝份额更小者出局。

Season 27 期间，评委分始终垫底的 Bobby Bones 夺冠引发争议。自 Season 28 起，节目引入 **Judge Save 规则**（定义如下），同时恢复 Rank 聚合机制。

> **定义 2 (Judge Save 规则，Season 28–34).** 在 Rank 聚合下计算 $C_i^{(t)}$。设 Bottom 2 为 $C_i^{(t)}$ 最大的两名选手，记为 $\mathcal{B} = \{b_1, b_2\}$。评委对 $\mathcal{B}$ 进行投票——模型中以评委分更低者出局模拟：$k = \arg\min_{i \in \mathcal{B}} S_i^{(t)}$。

粉丝票数据为商业秘密，其缺失使得定量分析赛制合理性与选手竞争力的核心因素极为困难。本研究聚焦以下四个问题：

> **Q1 (粉丝票重构).** 如何基于可观测数据 $\{S_i^{(t)}\}$ 和淘汰结果 $\{E_{\text{obs}}^{(t)}\}$，反推隐含的 $\{V_i^{(t)}\}$ 分布？  
> **Q2 (机制比较).** Rank 与 Percentage 两种聚合函数对比赛结果的影响是否存在系统性差异？  
> **Q3 (特征影响).** 人口学特征（年龄、行业）与舞伴选择是否异质地影响评委分与粉丝人气？  
> **Q4 (机制设计).** 能否设计一个在"技术公平"与"娱乐价值"之间实现 Pareto 改进的投票系统？

## 2. 模型假设与符号定义

### 2.1 基本假设

> **A1 (评委客观性).** 评委分 $S_i^{(t)}$ 客观反映选手 $i$ 在第 $t$ 周的技术水准，不存在与选手身份相关的系统性偏差。  
> **A2 (粉丝独立性).** 粉丝票 $V_i^{(t)}$ 仅受选手人气影响，与评委分 $S_i^{(t)}$ 无直接因果关系。  
> **A3 (时序连续性).** 粉丝支持度在相邻周之间具有惯性——$P_{F,i}^{(t)}$ 与 $P_{F,i}^{(t-1)}$ 高度相关，不存在突发性极端波动。  
> **A4 (淘汰确定性).** 淘汰结果仅由 $f(\mathbf{S}^{(t)}, \mathbf{V}^{(t)})$ 的规则决定，无其他场外干预因素。

### 2.2 核心符号

| 符号 | 含义 | 类型/值域 |
|:---|:---|:---|
| $S_i^{(t)}$ | 选手 $i$ 在第 $t$ 周的评委总分 | $\mathbb{R}_{\ge 0}$ |
| $V_i^{(t)}$ | 选手 $i$ 在第 $t$ 周的粉丝票数（**隐变量**） | $\mathbb{R}_{\ge 0}$ |
| $R_{J,i}$ | 评委分降序排名，$R_{J,i} = \text{rank}(-S_i^{(t)})$ | $\{1,\ldots,K\}$ |
| $R_{F,i}$ | 粉丝票降序排名，$R_{F,i} = \text{rank}(-V_i^{(t)})$ | $\{1,\ldots,K\}$ |
| $P_{J,i}$ | 评委分百分比，$P_{J,i} = S_i^{(t)} / \sum_j S_j^{(t)}$ | $(0,1)$ |
| $P_{F,i}$ | 粉丝票百分比，$P_{F,i} = V_i^{(t)} / \sum_j V_j^{(t)}$ | $(0,1)$ |
| $C_i$ | 组合指标（具体形式取决于聚合机制） | 见 Eq. (1)-(2) |
| $\alpha_{i,t}$ | Dirichlet 先验集中参数 | $\mathbb{R}_{>0}$ |
| $\lambda$ | 序贯贝叶斯更新的惯性权重 | $\mathbb{R}_{>0}$ |
| $w$ | 评委在组合指标中的权重 | $[0,1]$ |

def parse_elimination_week(result_str: object) -> Optional[object]:
    if pd.isna(result_str):
        return None
    text = str(result_str)
    match = re.search(r"Eliminated Week (\d+)", text)
    if match:
        return int(match.group(1))
    if "Place" in text:
        return "Final"
    return None

## 4. Task 1: 粉丝票估计模型

### 4.1 逆问题的形式化

粉丝票向量 $\mathbf{V}^{(t)}$ 从未公开，但下列信息已知：(a) 评委分 $\mathbf{S}^{(t)}$，(b) 淘汰结果 $E_{\text{obs}}^{(t)}$（即被淘汰选手的身份），(c) 聚合函数 $f$ 的精确计算逻辑。因此这是一个约束满足问题——寻找满足 $f(\mathbf{S}^{(t)}, \mathbf{V}^{(t)}) \implies E_{\text{obs}}^{(t)}$ 的所有 $\mathbf{V}^{(t)}$，构成可行解集合 $\mathcal{F}^{(t)}$。

### 4.2 三种投票机制的算法化定义

**机制 A: Rank-Based (Season 1–2)**

> **定义 3 (Rank 淘汰逻辑).** 设评委分向量 $\mathbf{S}$，粉丝排名向量 $\mathbf{R}_F$（$\mathbf{R}_F$ 为 $\{1,\ldots,K\}$ 的一个排列，1 为最受欢迎）。评委排名 $\mathbf{R}_J$ 由 $\text{rank}(-S_i, \text{method}=\text{'min'})$ 计算。组合得分：
>
> $$
> \boxed{C_i = R_{J,i} + R_{F,i}} \tag3
> $$
>
> 淘汰选手为 $k = \arg\max_i C_i$。若 $\exists i \neq j$ 使得 $C_i = C_j$，则取粉丝排名更差（值更大）者为 $k$。

**机制 B: Percentage-Based (Season 3–27)**

> **定义 4 (Percentage 淘汰逻辑).** 设评委分 $\mathbf{S}$，粉丝百分比 $\mathbf{P}_F$（$\sum_i P_{F,i}=1$）。评委百分比为 $P_{J,i} = S_i / \sum_j S_j$。组合得分：
>
> $$
> \boxed{C_i = P_{J,i} + P_{F,i}} \tag4
> $$
>
> 淘汰选手为 $k = \arg\min_i C_i$。若平局，取粉丝百分比更小者为 $k$。

**机制 C: Rank + Judge Save (Season 28–34)**

> **定义 5 (Judge Save 淘汰逻辑).** 首先按 Eq. (3) 计算 $C_i$。设 $\mathcal{B} = \{b_1, b_2\}$ 为 $C_i$ 最大的两名选手。淘汰选手 $k = \arg\min_{i \in \mathcal{B}} S_i$（评委分更低者）。

### 4.3 贝叶斯 MCMC 逆优化

将每周的粉丝支持建模为 $K$ 维概率单纯形上的 Dirichlet 隐变量：

$$
\boxed{\mathbf{P}_F^{(t)} = (P_{F,1}^{(t)}, \ldots, P_{F,K}^{(t)}) \sim \text{Dirichlet}(\boldsymbol{\alpha}^{(t)})} \tag5
$$

其中 $\boldsymbol{\alpha}^{(t)} = (\alpha_1^{(t)}, \ldots, \alpha_K^{(t)})$ 为集中参数向量。$\mathbf{P}_F^{(t)}$ 满足 $\sum_i P_{F,i}^{(t)} = 1$，$P_{F,i}^{(t)} > 0$。

为反映粉丝人气的时序惯性，构造自适应先验——将前一周的后验均值作为本周先验的信息来源：

$$
\boxed{\alpha_{i,t} = \alpha_0 + \lambda \cdot \hat{P}_{F,i,t-1}} \tag6
$$

其中：
- $\alpha_0 = 5.0$——基础集中参数，防止先验过强导致采样退化
- $\lambda = 100$——惯性权重，控制上周信息对本周先验的影响强度
- $\hat{P}_{F,i,t-1}$——选手 $i$ 在第 $t-1$ 周粉丝百分比的后验均值，由 MCMC 接受样本的经验均值估计

Eq. (6) 保证了：(i) 新选手（无历史 $\hat{P}$）回归均匀先验 $1/K$；(ii) 有历史选手的人气估计与上周保持连续性，符合现实中粉丝基础的"粘性"特征[^1]。

[^1]: Gelman A, Carlin J B, Stern H S, Dunson D B, Vehtari A, Rubin D B. *Bayesian Data Analysis*, Chapman and Hall/CRC, 3rd Edition, 2013.

**后验估计** 通过拒绝采样实现（Algorithm 1）：

```
Algorithm 1: Sequential MCMC for Fan Vote Reconstruction
═══════════════════════════════════════════════════════════
Input:  Season s, all weekly data {(S^(t), E_obs^(t))} for s
Output: Posterior mean P̂_F^(t) and std σ_F^(t) for each week
Hyperparams: N = 50,000, α_0 = 5.0, λ = 100

1  prior ← {}   // 赛季初无先验信息
2  for each week t ∈ sorted_weeks(s) do
3      mode ← 'rank' if s ∈ {1,2} ∪ {28..34} else 'percent'
4      for each contestant i in week t do
5          if i ∈ prior then
6              α_i ← α_0 + λ · prior[i]          // 含惯性
7          else
8              α_i ← α_0 + λ · (1/K)             // 均匀先验
9          end if
10     end for
11     accepted ← []
12     for iter = 1 to N do
13         p_F ~ Dirichlet(α)
14         if mode == 'rank' then
15             r_F ← rankdata(-p_F, method='min')
16             elim ← rank_elimination(S^(t), r_F)   // 按 Eq. (3)
17         else
18             elim ← percent_elimination(S^(t), p_F) // 按 Eq. (4)
19         end if
20         if elim ∈ elim_indices(E_obs^(t)) then
21             accepted.append(r_F if mode == 'rank' else p_F)
22         end if
23     end for
24     P̂_F^(t) ← mean(accepted, axis=0)
25     σ_F^(t) ← std(accepted, axis=0)
26     prior ← P̂_F^(t)   // 传递到下周
27 end for
```

### 4.4 核心实现

```python
def simulate_fan_votes(season_num, week_num, num_samples=50000,
                        prior_support=None):
    subset = df_weekly[(df_weekly['Season'] == season_num) &
                       (df_weekly['Week'] == week_num)]

    contestants = subset['Contestant'].values
    judge_scores = subset['JudgeScore'].values
    mode = 'rank' if (season_num <= 2 or season_num >= 28) else 'percent'
    num_c = len(contestants)

    # Dirichlet 集中参数 (Eq. 6)
    base_alpha, inertia_weight = 5.0, 100.0
    alphas = []
    for name in contestants:
        if prior_support and name in prior_support:
            prev = prior_support[name]
            if mode == 'rank' and prev >= 1.0:
                weight = (num_c + 1 - prev)       # 排名 → 伪权重
                alphas.append(base_alpha + 5.0 * weight)
            else:
                alphas.append(base_alpha + inertia_weight * prev)
        else:
            alphas.append(base_alpha + inertia_weight * (1.0 / num_c))

    alphas = np.array(alphas)
    raw_samples = np.random.dirichlet(alphas, num_samples)

    # 拒绝采样
    valid_fan_votes = []
    for fan_vec in raw_samples:
        if mode == 'rank':
            fan_ranks = rankdata([-p for p in fan_vec], method='min')
            loser_idx = calculate_rank_elimination(judge_scores, fan_ranks)
        else:
            loser_idx = calculate_percent_elimination(judge_scores, fan_vec)

        if loser_idx in elim_indices:
            valid_fan_votes.append(fan_ranks if mode == 'rank' else fan_vec)

    return np.array(valid_fan_votes), contestants, mode
```

> **注.** Rank 模式下，由于后验均值 $\hat{P}_{F,i,t-1}$ 存储的是排名值（$\ge 1$），在构造先验时需通过 `weight = (num_c + 1 - prev)` 将其线性变换为伪权重——排名越优（数值越小），权重越高，与 Dirichlet 集中参数的语义一致。

### 4.6 模型诊断与验证

#### 4.6.1 后验方差分析

估计不确定性 $\sigma_{F,i}$ 反映模型解的唯一性：

$$
\boxed{\sigma_{F,i} = \sqrt{\frac{1}{|\mathcal{F}_i|-1} \sum_{m=1}^{|\mathcal{F}_i|} (P_{F,i}^{(m)} - \hat{P}_{F,i})^2}} \tag7
$$

- **低方差 ($\sigma < 0.5$)**：淘汰结果对粉丝票有**强约束**，存在唯一的粉丝票分布可解释淘汰
- **高方差 ($\sigma > 1.0$)**：约束弱，多见于评委分差异较小的场景，多种粉丝票配置均可行

![各赛季粉丝票估计不确定性](/figures/2026MCM_C/diagnostics.png)

#### 4.6.2 预测效度 (Spearman 秩相关)

以最终排名 $\text{Placement}_i$（1 = 冠军）为外部校标，验证估计粉丝支持 $\hat{P}_{F,i}$ 的预测效度：

$$
\boxed{\rho = 1 - \frac{6 \sum_i d_i^2}{n(n^2 - 1)}} \tag8
$$

其中 $d_i$ 为选手 $i$ 的估计粉丝支持排名与最终排名之差，$n$ 为选手数。

| 模式 | Spearman $\rho$ | $p$ 值 | 预期符号 | 结论 |
|:---|:---:|:---:|:---|:---|
| Rank | $+0.236$ | $< 0.01$ | 正（高排名值 = 差表现） | 符合预期 |
| Percentage | $-0.457$ | $< 0.001$ | 负（高百分比 = 好表现） | 符合预期 |

#### 4.6.3 灵敏度与稳健性

**惯性权重灵敏度检验：** 对代表性周次 (Season 19, Week 5)，在 $\lambda \in \{10, 100, 500\}$ 下重复运行 Algorithm 1，记录 $\hat{P}_{F,i}$ 的相对变化。三个水平下的估计值在 $\pm 3\%$ 范围内收敛。

**评委分扰动稳健性检验：** 对评委分施加 multiplicative noise：

$$
\boxed{\tilde{S}_i^{(t)} = S_i^{(t)} \cdot (1 + \varepsilon_i), \quad \varepsilon_i \sim \text{Uniform}(-0.20, +0.20)} \tag9
$$

在 20% 噪声水平下，估计粉丝份额的平均波动为 **4.36%**，对同一周次 (Season 19, Week 5) 的变化仅为 **0.5%**，满足稳定性要求（$|\Delta \hat{P}_F| < 5\%$）。

### 4.7 小结

$$
\boxed{\text{Feasibility Rate} = \frac{259}{264} = 98.5\%} \tag10
$$

统一隐变量模型在 34 个赛季的 264 个淘汰事件中成功再现了 259 个。失败的 5 个案例集中于数据稀疏的早期赛季（参赛者少、约束弱导致采样效率低）。正则化参数 $\lambda \in [10, 500]$ 内估计收敛，20% 噪声扰动下粉丝份额变化 $< 1\%$，模型稳健性通过验证。

## 5. Task 2: 投票机制对比分析

### 5.1 聚合函数的数学性质

将两种聚合机制统一到连续隐支持度空间，分析其结构差异。

**Rank 法** 的组合指标 $C_i^{\text{rank}} = R_{J,i} + R_{F,i}$ 的取值集合为 $\{2, 3, \ldots, 2K\}$，是一个离散有限集。其核心性质是**序数不变性**——$R(\cdot)$ 对自变量的任意严格递增变换保持不变：

$$
\boxed{R(x) = R(g(x)), \quad \forall g: g'(x) > 0} \tag11
$$

这意味着两名评委分相差 10 分与相差 1 分的选手，在 Rank 空间中的差距可能仅为一个名次。此"排名压缩"效应导致：即使粉丝基础极度极化，最大粉丝优势被限制为 $K-1$ 个名次差。

**Percentage 法** 的组合指标 $C_i^{\text{pct}} = P_{J,i} + P_{F,i}$ 的值域为 $(0, 2)$ 上的连续区间。其核心性质是**仿射协变性**——票数的绝对值差异完整保留在百分比中。考虑粉丝票对组合指标的边际效应：

$$
\boxed{\frac{\partial C_i^{\text{pct}}}{\partial P_{F,i}} = 1, \quad \forall i} \tag12
$$

即粉丝百分比每增加 1 个百分点，组合指标恰好增加 1 个百分点——不存在 Rank 法中的"上限"约束。当一个选手的粉丝票集中度 $\frac{V_i}{\sum V} \to 1$ 时，$P_{F,i} \to 1$，足以单独覆盖任何评委分劣势。

**跨方法可比性：** 将 Rank 法的排名映射回连续尺度。在均匀粉丝票假设下，从百分比到排名的信息损失约为：

$$
\boxed{\Delta I \approx \log_2 K \ \text{bits}} \tag13
$$

$K$ 越大（赛季早期），Rank 法的信息损失越大，粉丝票对组合指标的影响力相对更弱。

### 5.2 规则灵敏度评估

对每个淘汰周 $(s, t)$，分别以三套规则重演淘汰逻辑：**(1) Percent-Sum**（Eq. 4），**(2) Rank-Sum**（Eq. 3），**(3) Judge Save**（定义 5）。定义两个评估指标：

$$
\boxed{\text{Match Rate} = \frac{\#\{\text{predicted elim} = \text{actual elim}\}}{\#\text{elimination weeks}}} \tag14
$$

$$
\boxed{\text{Disagreement Rate} = \frac{\#\{\text{Rule A} \neq \text{Rule B}\}}{\#\text{elimination weeks}}} \tag15
$$

三套规则的成果展示为论文 Figure 3（匹配率柱状图）与 Figure 4（配对分歧率热力图），核心结论：

- Percent-Sum 与 Rank-Sum 的分歧率在 Season 3–27（Percentage 时代）显著高于其他时期——因为原始数据由 Percentage 规则生成，Rank 替代规则更可能产生不同结果
- Judge Save 规则与纯 Rank-Sum 的分歧集中在赛季晚期（$K \le 5$），此时 Bottom 2 占总人数比例大，Judge Save 介入频率高

### 5.3 "粉丝救援"阈值

为量化"低评委分 + 高粉丝票 = 存活"这一现象，定义标准化相对人气指数：

> **定义 6 (Relative Popularity Index, RPI).** 对选手 $i$ 在第 $t$ 周，计算其粉丝支持度在该周内的 Z-score：
> $$
> \boxed{\text{RPI}_i^{(t)} = \text{sign}(\text{mode}) \cdot \frac{\hat{P}_{F,i}^{(t)} - \bar{P}_F^{(t)}}{\sigma_{P_F}^{(t)}}} \tag16
> $$
>
> 其中 $\text{sign}(\text{mode}) = -1$（Rank 模式，因低排名 = 好人气）或 $+1$（Percent 模式），$\bar{P}_F^{(t)}$ 与 $\sigma_{P_F}^{(t)}$ 分别为该周粉丝支持的均值和标准差。RPI > 0 表示该选手比该周平均值更受欢迎。

Split Violin Plot（论文 Figure 6）揭示了清晰的分叉现象：在评委分 20-30 区间，Safe 组（绿）的 RPI 中位数 $\approx +0.8$，Eliminated 组（红）的 RPI 中位数 $\approx -0.5$。Mann-Whitney U 检验在 $\alpha = 0.01$ 水平下显著。**RPI > 1.0 构成"粉丝救援"的充分条件**——无论评委分多低，此类选手几乎从未被淘汰。

## 6. Task 3: 争议案例模拟与 Judge Save 影响

### 6.1 反事实模拟的实验设计

> **定义 7 (反事实模拟).** 对争议案例 $(s, t)$，使用模型估计的后验均值 $\hat{\mathbf{P}}_F^{(t)}$ 作为"真实"粉丝票，代入三套不同规则重演淘汰逻辑。记录每种规则下的淘汰者，并与历史实际淘汰者比较。Judge Save 规则以定义 5 实现。

```python
def simulate_week_rule(season, week, method='rank', judge_save=False):
    week_data = est_df[(est_df['Season'] == season) &
                       (est_df['Week'] == week)]
    names = week_data['Contestant'].values
    j_scores = week_data['Judge_Score'].values
    f_support = week_data['Est_Fan_Support'].values

    if method == 'rank':
        j_ranks = rankdata([-s for s in j_scores], method='min')
        f_ranks = rankdata([-f for f in f_support], method='min')
        combined = j_ranks + f_ranks                      # Eq. (3)
    else:
        j_pcts = j_scores / np.sum(j_scores)
        f_pcts = f_support / np.sum(f_support)
        combined = j_pcts + f_pcts                        # Eq. (4)

    res_df = build_results_dataframe(names, j_scores, f_support,
                                      combined, method)

    if judge_save:
        bottom_2 = res_df.iloc[:2]                        # 定义 5
        final_loser = (bottom_2.iloc[0]
                       if bottom_2.iloc[0]['JudgeScore'] <
                          bottom_2.iloc[1]['JudgeScore']
                       else bottom_2.iloc[1])
        return final_loser['Contestant'], res_df

    return res_df.iloc[0]['Contestant'], res_df
```

### 6.3 案例分类与机制性解释

#### 第 I 类：超级多数选手 (Jerry Rice, Bobby Bones)

这两名选手的粉丝支持远超同期竞争对手。设其粉丝排名 $R_F^* \approx 1$，评委排名 $R_J^* = K$（垫底），则 Rank 组合分为：

$$C^* = 1 + K$$

当其他选手的粉丝排名分散（如 $\{2,3,\ldots,K\}$）时，其组合分 $C_j = R_{J,j} + R_{F,j}$ 可能超过 $1+K$，因此 $C^*$ 并非必然最大。在我们的模拟中，**Jerry Rice 在 S2-W5 的 $C^* = 1+6 = 7$，而 George Hamilton 的 $C = 2+5 = 7$（平局，Tia Carrere 的 $C=3+4=7$），按粉丝排名决胜后 George 出局**。Bobby Bones 同理。

Judge Save 机制对这类选手**完全无效**——因为其粉丝排名 Rank 1 使 $C^*$ 不够大，根本不会落入 Bottom 2。Judge Save 的触发条件（Bottom 2）在此被高粉丝排名所阻断。

#### 第 II 类：边际争议选手 (Billy Ray Cyrus)

Billy Ray 的粉丝支持 $\approx 15.9\%$ 足以支撑其存活至中后期，但不足以完全避开 Bottom 2。在 Judge Save 模拟中，Billy Ray 落入 Bottom 2，评委据此比较其与另一 Bottom 2 选手的评委分。模拟淘汰 Billy Ray，恢复了 meritocracy。**此案例证明了 Judge Save 在"温和争议"区间的有效性。**

#### 第 III 类：混沌案例 (Bristol Palin)

Bristol Palin 的粉丝分布恰好处于两个聚合函数的"断点"附近——Rank 法与 Percentage 法产生了不同的淘汰者。Rank 法下，排名压缩效应使得评委高分选手 Jennifer Grey 的 $R_{J,\text{Grey}} = 1$ 被其他选手的粉丝排名优势所淹没；Percentage 法下，Bristol 的粉丝百分比优势被完整保留。此案例说明**规则选择可系统性地改变比赛走向**，即使对同一组选手和粉丝票。

## 7. Task 4: 影响因素分析

### 7.1 模型设定

两个互补的回归模型用于量化外生特征的影响。

**模型 1: Logistic 回归（淘汰风险）**
$$
\boxed{\text{logit}(P(Y_{it}=1)) = \beta_0 + \beta_1 \cdot Z_{\text{score},it} + \beta_2 \cdot Z_{\text{fan},it} + \boldsymbol{\gamma}^\top \mathbf{X}_i} \tag17
$$
其中 $Y_{it} \in \{0,1\}$ 为二值淘汰标签，$Z_{\text{score}}, Z_{\text{fan}}$ 为标准化评委分与粉丝支持度，$\mathbf{X}_i$ 包含行业、舞伴、年龄的 One-Hot 编码。系数 $\beta_j$ 的解释为：**该变量每增加 1 个标准差，淘汰几率 (odds) 乘以 $e^{\beta_j}$。**

**模型 2: OLS 回归（粉丝人气驱动因素）**
$$
\boxed{Z_{\text{fan},it} = \alpha + \boldsymbol{\delta}^\top \mathbf{X}_i + \varepsilon_{it}} \tag18
$$
**模型 3: OLS 回归（评委分驱动因素）**
$$
\boxed{Z_{\text{score},it} = \alpha + \beta_1 \cdot \text{Age}_{std} + \beta_2 \cdot \text{Age}_{std}^2 + \boldsymbol{\delta}^\top \mathbf{X}_i + \varepsilon_{it}} \tag19
$$
年龄的二次项 $\text{Age}_{std}^2$ 用于捕捉潜在的非线性生物学或经验相关的评分效应。

### 7.2 评委分驱动因素（技术偏好）

以 Eq. (19) 建模，OLS 回归结果（论文 Figure 8, Figure 9）：

**年龄效应：** 线性系数 $\beta_1 = -0.40$ ($p < 0.0001$)，二次项 $\beta_2$ 较小但显著，整体呈单调递减后趋于平稳的趋势。20-35 岁青年选手得分最高；50 岁以上选手的评分衰减趋于平缓——评委可能对经验丰富的年长舞者给予一定认可。
$$
\boxed{\frac{\partial \mathbb{E}[\text{Score}]}{\partial \text{Age}} \approx -0.40 + 2\beta_2 \cdot \text{Age}_{std} < 0 \ \text{for Age} > 25} \tag20
$$
**行业效应：** 以"Other"行业为基准组，舞者与演员的评委分分别高出约 1.5-2 分，体育明星中性偏高，社交媒体名人高出约 $+0.49$。

**舞伴效应：** 顶级职业舞伴（如 Derek Hough）与明星搭档的评委分系统性偏高（$\beta \approx +0.45$，$p < 0.05$），验证了专业指导对技术表现的促进作用。

### 7.3 粉丝支持驱动因素（观众偏好）

**年龄的 U 形（确切地说是倒 U 形 + 右尾反弹）：** 一次项 $\beta = -0.09$ (n.s.)，二次项显著，三次多项式拟合揭示人气峰值出现在 **49.7 岁**。年轻选手享受"新鲜感溢价"，60 岁以上"传奇人物"享受"怀旧溢价"（Tommy Chong 效应），中间年龄段的选手相对缺乏粉丝吸引力。

\boxed{\mathbb{E}[Z_{\text{fan}} \mid \text{Age}] \approx \gamma_0 + \gamma_1 \text{Age} + \gamma_2 \text{Age}^2 + \gamma_3 \text{Age}^3, \quad \text{peak at Age} \approx 49.7} \tag21
$$
**行业效应的核心发现：**

| 行业 | 评委系数 (技术) | 粉丝系数 (人气) | 粉丝/评委比 |
|:---|:---:|:---:|:---:|

## 8. Task 5: 机制设计优化

### 8.1 问题形式化

定义广义投票机制的组合指标为评委分与粉丝票的加权和：
$$
\boxed{C_i(w) = w \cdot \tilde{S}_i + (1-w) \cdot \tilde{V}_i} \tag22
$$
其中 $\tilde{S}_i, \tilde{V}_i$ 分别为归一化（百分比）评委分与粉丝票，$w \in [0, 1]$ 为评委权重。$w=0.5$ 对应传统的 50/50 Percentage 法；$w \to 1$ 偏向纯评委决策；$w \to 0$ 偏向纯粉丝决策。

定义两个竞争性的优化目标：

> **定义 8 (False Elimination Rate, FER).** 评委分 Top 3 的选手被淘汰的概率：
>
> $$
> \boxed{\text{FER}(w) = \frac{\#\{(s,t): \text{Top3}_{\text{judge}} \cap \text{Eliminated}(w) \neq \varnothing\}}{\#\text{elimination weeks}}} \tag23
> $$
>
> 理想目标：$\text{FER} \to 0$。

> **定义 9 (Retention Rate, RR).** 粉丝人气 Top 3 的选手存活的概率：
>
> $$
> \boxed{\text{RR}(w) = \frac{\#\{(s,t): \text{Top3}_{\text{fan}} \cap \text{Eliminated}(w) = \varnothing\}}{\#\text{elimination weeks}}} \tag24
> $$
>
> 理想目标：$\text{RR} \to 1$。

### 8.2 多目标网格搜索

在历史数据上以步长 $\Delta w = 0.05$ 遍历 $w \in [0, 1]$，计算每个 $w$ 下的 FER 和 RR。结果（论文 Figure 12）：

- $w \uparrow \implies \text{FER} \downarrow$（更好 meritocracy）但 $\text{RR} \downarrow$（有丢失粉丝最爱的风险）
- $w \downarrow \implies \text{RR} \uparrow$（更好 retention）但 $\text{FER} \uparrow$（产生技术惨案）
- Pareto 前沿上的静态最优解位于 $w^* \approx 0.45$

### 8.3 Dynamic Phase-Shift 策略

单个静态权重在赛季全程上并非最优——赛季早期与晚期的目标函数存在本质差异。因此提出分阶段动态策略：
$$
\boxed{w(t) = \begin{cases}
0.10, & 1 \le t \le 8 \quad \text{(Engagement Phase)} \\
0.45, & 9 \le t \le T \quad \text{(Championship Phase)}
\end{cases}} \tag25
$$

| 阶段 | 周数 | 评委权重 | 粉丝权重 | 机制目标 |
|:---|:---|:---:|:---:|:---|
| **Phase 1 (Engagement)** | Week 1–8 | 0.10 | 0.90 | 最大化观众留存，保护多样化选手角色 |
| **Phase 2 (Championship)** | Week 9–决赛 | 0.45 | 0.55 | 向 meritocracy 倾斜，确保冠军技术含金量 |
| **Technical Safety Net** | 全赛季 | — | — | 若 Top 3 技术分选手落入 Bottom 3，触发评委拯救 |

Safety Net 规则的形式化定义：

> **定义 10 (Technical Safety Net).** 设 $\mathcal{T}^{(t)} = \{i: S_i^{(t)} \in \text{Top3}\}$ 为技术前三。按 Eq. (22) 计算 $C_i$ 后，设 $\mathcal{B}_3^{(t)}$ 为 Bottom 3。若 $\mathcal{T}^{(t)} \cap \mathcal{B}_3^{(t)} \neq \varnothing$，则评委从 $\mathcal{B}_3^{(t)}$ 中淘汰一位非技术前三的选手（评委分最低者）。此规则确保技术顶尖选手不会被投票淘汰。

### 8.4 性能对比

| 机制 | FER | RR | 综合效用 |
|:---|:---:|:---:|:---|
| **Static 50/50** | $\approx 5.2\%$ | $\approx 85\%$ | 基准 |
| **Static 45/55** | $\approx 3.1\%$ | $\approx 88\%$ | Pareto 最优静态点 |

**To:** Producers of Dancing with the Stars  
**From:** Team 2613579  
**Date:** February 2, 2026  
**Subject:** Analysis of Voting Mechanisms and Recommendations for Future Seasons

### Executive Summary

我们对 34 季 *Dancing with the Stars* 数据进行了综合分析。贝叶斯 MCMC 模型以 **98.5%** 的可行率再现了历史淘汰结果，Rank 与 Percentage 聚合函数的系统性比较揭示了两种机制在公平性维度上的结构性差异，并基于多目标优化提出了一种动态分阶段投票策略。

### Key Findings

**1. 粉丝救援阈值。** 当选手的 Relative Popularity Index (Z-score) > 1.0 时，低评委分几乎不会导致淘汰。粉丝支持是淘汰决策中的决定性安全网。

**2. 投票机制的本质差异。**
- **Percentage-Based** — 核心：$C_i = P_{J,i} + P_{F,i}$。结构性偏向极化粉丝团体，允许争议性冠军（Bobby Bones 类案例）。
- **Rank-Based** — 核心：$C_i = R_{J,i} + R_{F,i}$。排名压缩贬低估评高分，对温和离群值有效，但对超级粉丝多数约束不足。
- **Judge-Save** — 核心：Bottom 2 评委投票。仅在选手实际落入 Bottom 2 时触发，对 Billy Ray Cyrus 类案例高效，对 Bobby Bones 类案例完全无效。

**3. 评委与粉丝偏好的系统性分歧。**

| 因素 | 评委偏好 | 粉丝偏好 | 分歧特征 |
|:---|:---|:---|:---|
| 年龄 | 线性递减 ($\beta = -0.40$, $p < 0.001$) | 三次 U 形 (峰值 49.7 岁) | 粉丝给予 60+ 岁"怀旧溢价" |
| 社交媒体红人 | $+0.49$ | **$+2.53$** ($p < 0.001$) | 粉丝优势 = $6\times$ 技术优势 |
| 舞伴 | 技术专家显著正面 | 仅弱相关 ($r=0.18$) | 技术不等于人气 |

### Recommendations

1. **Phase 1 (Week 1–8):** $w_{\text{judge}} = 0.10$（10% 评委 / 90% 粉丝），最大化观众参与
2. **Phase 2 (Week 9–Finals):** $w_{\text{judge}} = 0.45$（45% 评委 / 55% 粉丝），确保冠军技术水准
3. **Technical Safety Net:** Top 3 技术分选手落入 Bottom 3 时自动触发评委拯救

**预期效果：** False Elimination Rate 从 $5.2\% \to 0.0\%$；Retention Rate 提升至 $91.8\%$，两项指标均超过任何静态权重分割。

2. 根据赛季总周数调整阶段转换点（建议第 9 周）
3. 向观众与评委清晰传达新规则以保障透明度

## 10. 方法论反思

### 10.1 假设与局限

**局限 1: 逆推的唯一性。** 对许多淘汰周次，粉丝票的可行解区域并非单点——不同粉丝分布可产生相同淘汰结果。后验方差 $\sigma_{F,i}$ 量化了此不确定性。Rank 模式下约束较紧（$\bar{\sigma}$ 较小），Percentage 模式下可行域更宽。

**局限 2: Season 28+ 的 Judge Save。** 批量 MCMC 重构中假设淘汰者 = 最低组合分者，这忽略了 Judge Save 中 Bottom 2 + 评委投票的机制。对此，我们在 Section 6 进行了独立的逐案例反事实模拟作为补充。

**局限 3: Dirichlet 的独立同分布假设。** Dirichlet 分布假设 $\mathbf{P}_F$ 各分量满足负相关约束 $\sum P_{F,i} = 1$，但未显式建模粉丝之间的"转移"现象（某选手淘汰后，其粉丝可能系统性转向特定选手）。

**局限 4: 舞伴选择偏差。** OLS 回归中的舞伴系数包含不可忽略的选择偏差——高知名度舞伴更可能被分配给高潜力明星。我们报告的系数应解读为关联性而非因果性。

### 10.2 技术启示

**Informative Prior 的本质性作用。** 平坦 Dirichlet 先验 ($\boldsymbol{\alpha} = \mathbf{1}$) 下可行率仅约 60%。引入 Eq. (6) 的序贯贝叶斯更新使可行率跃升至 98.5%。这说明了高维隐变量推断中的一条原则：**在先验信息极度匮乏时，均匀采样几乎无法命中与约束一致的区域。通过时序结构注入 Informative Prior 将推断从不可能变为可能。**

**反事实模拟对直觉的修正。** Bobby Bones 案例是最具方法论启发性的发现。普遍舆论认为"恢复 Rank 法即可防止 Bobby Bones 问题"，但形式化模拟表明 Eq. (3) 在 $R_F^* = 1, R_J^* = K$ 时 $C^* = 1+K$，当其他选手排名分散时并非必然最大。**形式化建模在此纠正了一个广泛共享的错误直觉。**
